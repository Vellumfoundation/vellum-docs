# Upgrade Governance

Upgrade governance controls protocol and bridge changes.

## Recommended process

1. Draft upgrade proposal.
2. Identify affected contracts and services.
3. Publish risk assessment.
4. Test on devnet.
5. Test on testnet.
6. Collect approvals.
7. Execute during a monitored window.
8. Verify and publish completion.

{% hint style="warning" %}
Users should know whether an upgrade can affect balances, withdrawals, bridge routing, or transaction inclusion.
{% endhint %}
